# Provisional EPLI Slice

This note documents the permanent exploratory EPLI slice asset added for
notebook-side HFO testing.

## Purpose

This asset is for **mechanism exploration**, not biological canonization. It is
good enough to test whether adding one fast EPL inhibitory population can help
produce or stabilize the target HFO regime. It is not good enough to treat a
positive result as validation of the real olfactory bulb circuit.

## Slice Asset

- Slice name: `DorsalColumnSliceEPLIProvisional`
- Source slice: canonical `DorsalColumnSlice`
- New group: `EPLIs`
- Cell model: `SyntheticEPL2026.PVCRH_FSI1`
- Cell count: `24`

## Placement Rule

- One EPLI is placed near each canonical TC soma.
- The EPLI soma is offset tangentially by `12 um` from the paired TC soma.
- This avoids literal soma overlap while preserving strong local TC contact in
  the maintained full canonical slice geometry.

## Synapse Rules

### `EPLIs__TCs`

- reciprocal set
- source pattern: `*dend*`
- target pattern: `*soma*`
- max distance: `10 um`
- use radius: `True`
- max synapses per point: `1`

Observed exported metrics:

- entries: `71`
- EPLI source coverage: `1.0`
- TC target coverage: `1.0`
- median gap distance: `~4.96 um`

Interpretation:

- this is the cleaner first-pass fast-loop test asset
- it is TC-focused
- it is the better first notebook experiment if the question is simply whether
  a fast EPLI loop can support HFO-like dynamics

### `EPLIs__MCs`

- reciprocal set
- source pattern: `*dend*`
- target pattern: `*dend*`
- max distance: `8 um`
- use radius: `True`
- max synapses per point: `1`

Observed exported metrics:

- entries: `85`
- EPLI source coverage: `0.75`
- MC target coverage: `1.0`
- median gap distance: `~5.26 um`

Interpretation:

- this is broader and less clean than the TC set
- it exists for exploratory M/T-wide tests
- it should not be the first mechanism claim if the TC loop alone already
  explains the effect

## Notebook Paramsets

- `GammaSignature_EPLI_Provisional_TCOnly`
- `GammaSignature_EPLI_Provisional_MTC`

Recommended first run:

- `GammaSignature_EPLI_Provisional_TCOnly`

Reason:

- it isolates the fast TC/EPLI loop more cleanly
- it matches the present geometry evidence better than assuming symmetric MC and
  TC importance

## Regeneration

The asset is generated by:

```bash
MPLCONFIGDIR=/tmp/mpl /opt/miniconda3/envs/OBGPU/bin/python \
  tools/build_provisional_epli_slice.py
```

That script copies the canonical slice, inserts the synthetic EPLI population,
and writes the reciprocal `EPLIs__TCs` and `EPLIs__MCs` JSON files.
